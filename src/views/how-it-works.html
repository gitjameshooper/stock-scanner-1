How it Works

Main Steps

  • Authentication/Symbol Prep. - Authorize use of TradeKing.com API
  • Tier Process - Tier system to run through all stock symbols
  • Stock Tests - Runs all stocks through tests
  • Buckets - Put stocks in buckets for best trades

Authentication/Symbol Prep.

  • Uses oAuth to authorize
  • Makes JSON call to get public and secret keys/tokens
  • Makes JSON call to get all stock symbols
  • Formats symbols into a string and start the tier process

Tier Process

  • Makes a call to TradeKing's REST API every second(api limit 1 call/sec.) to get data for 350 stocks.
  • Afer each call it puts the 350 stocks through different tests.

Stock Tests

  • Using these data points(under the data fields section) tests are run to determine if the stock fits a certain criteria
  • Volume Test

    This a volume tier process where it makes sure the cumlative volume is above a certain threshold on the hour every hour.

    All stocks go through this test first. If it passes then it moves to the next.
  • ABCD Test

    This test consists of three smaller tests

  • VWAP Test

    Checks to see how far

  • If the stock passes all the tests then it gets added to a bucket.
  • Then it makes another API call for 350 more stocks.

Buckets

  • Once it makes through all the stocks it then applies the buckets of data to the view.(avg. 10 secs.)
  • Three Main Buckets
    • ABCD
    • VWAP
    • Fast Moving
  • Then the Tier Process starts over again and runs through all the stock symbols

Using the App

  • Start/Stop Buttons - Make the app start/stop scanning the stocks
  • Green Border - Scanner is working
  • Red Border - Scanner has stopped
  • Blue Border - There is an error
  • Above each bucket you have variables you can change to update your scanning paramaters
